Smart Baby Care

IoT Monitoring & AI Guardian System

The Challenge

Why Smart Monitoring Matters

SIDS Risk Factors

The prone (face-down) sleep position significantly increases the risk of Sudden Infant Death Syndrome. Constant manual monitoring is impossible for parents.

Delayed Response

Traditional baby monitors simply transmit audio. They don't analyze context, differentiate between sounds, or offer immediate mitigation strategies.

Our Solution

An intelligent IoT framework utilizing edge computing and cloud AI to proactively detect distress (crying), analyze dangerous postures, and execute automated physical responses.

IoT Edge Layer

Hardware Schematic

Control Node

ESP32 DevKit V1
  • INMP441 (I2S Mic): Digital audio capture
  • MAX98357A (DAC): Streaming TTS playback
  • DHT11: Room environment sensing
  • Relay: Fan control
  • SG90 Servo: Cradle swing mechanism
protocol: WebSocket (Port 8000)
power: 5V DC

Vision Node

ESP32-CAM
  • OV2640 Sensor: 2MP Camera module
  • Live Stream: Continuous MJPEG relay
  • HQ Capture: High-res JPEG burst mode
  • Memory: 4MB PSRAM for frame buffers
protocol: HTTP POST/GET (Port 8003)
architecture: Dual-board separation
Why two boards? Audio requires 2 I2S ports, while Vision requires extensive GPIO footprint. Separation ensures zero processing bottlenecks.

Core Pipeline 1

AI Cry Detection & Analysis

1. Edge Trigger

RMS Threshold

ESP32 measures Root Mean Square every 500ms. If RMS > threshold, it captures 5s of PCM audio.

2. VAD & ASR

Silero & Groq Whisper

Silero filters non-voice. Groq's high-speed Whisper model transcribes intent.

3. Intent & Alert

Event Dispatch

Server validates cry, triggers ESP32-CAM snapshot, and fires Telegram interactive alert with fallback 60s cooldown.

Gemini Vision Integration

Leveraging Google's advanced visual LLMs to determine dangerous sleep positions inherently linked to SIDS risk.

PRONE (Face-down) SUPINE (Safe)

Stream Pre-emption

ESP32-CAM has limited memory footprint. The server explicitly executes `drop_all_streams()` to halt MJPEG feeds, ensuring maximum RAM availability for the high-res HQ snapshot needed by Gemini AI.

System Highlights

Performance Optimizations

Persistent WebSockets

Replaced traditional HTTP polling with lightweight WebSockets. Minimizes ESP32 battery consumption and provides structural sub-100ms bidirectional latency.

Native MJPEG Relay

Server relays raw frame bytes using `multipart/x-mixed-replace`. This bypasses complex RTSP encoding requirements on the low-power ESP32.

Vietnamese NLP

Dual conversational layer handling robust natural text intent parsing mapping vague parent requests to hard hardware commands.
